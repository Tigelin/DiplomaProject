# 📓 Электронный журнал Приморского политехнического колледжа

Система для учёта успеваемости и посещаемости студентов. Проект разработан в рамках преддипломной практики.

---

## 🚀 Описание проекта

Данный веб-сервис автоматизирует процессы учета успеваемости, ведения журналов и координации учебного процесса в колледже.

### Основные возможности

* 🗓️ **Расписание занятий:** Просмотр актуального расписания с удобной фильтрацией по учебным группам.
* 👥 **Ролевая модель:** Личные кабинеты с разграничением прав для студентов, преподавателей и администраторов.
* 📊 **Успеваемость и посещаемость:** Удобный интерфейс выставления/просмотра оценок и ведения учета посещаемости.
* 📝 **Управление заданиями:** Создание, редактирование и выдача учебных заданий.
* 📁 **Интерактивные материалы:** Прикрепление файлов к занятиям (включая автоматический предпросмотр изображений).
* 📥 **Экспорт данных:** Выгрузка журналов оценок и отчетов в форматы **Excel** (`.xlsx`) и **Word** (`.docx`).
* 🔍 **Поиск и фильтрация:** Быстрый поиск по всем табличным данным.
* 📱 **Адаптивный интерфейс:** Полноценная поддержка мобильных устройств и планшетов благодаря **Bootstrap 5**.

## 💻 Установка и запуск проекта

### 1. Клонирование репозитория

```bash
git clone https://github.com/Tigelin/DiplomaProject.git
cd college-journal

```

### 2. Настройка виртуального окружения

```bash
# Создание окружения
python -m venv venv

# Активация (Windows)
venv\Scripts\activate

# Активация (Linux/Mac)
source venv/bin/activate

```

### 3. Установка зависимостей

```bash
pip install --upgrade pip
pip install -r requirements.txt

```

### 4. Конфигурация переменных окружения

Создайте файл `.env` в корневой директории проекта по образцу `.env.example` и заполните его своими данными:

```env
DEBUG=True
SECRET_KEY=your_secret_key_here
DATABASE_URL=postgresql://journal_user:12345678@localhost:5432/mydb2
```

### 5. Настройка базы данных PostgreSQL

Зайдите в консоль PostgreSQL (`psql` или через pgAdmin) и выполните следующие SQL-команды для создания БД и пользователя:

```sql
CREATE DATABASE diploma_database;
CREATE USER journal_user WITH PASSWORD '12345678';
ALTER ROLE journal_user SET client_encoding TO 'utf8';
ALTER ROLE journal_user SET timezone TO 'Asia/Vladivostok';
GRANT ALL PRIVILEGES ON DATABASE diploma_database TO journal_user;
GRANT ALL ON SCHEMA public TO journal_user;

```

### 6. Восстановление базы данных из дампа

Импортируйте структуру и тестовые данные из SQL-дампа:

```bash
psql -U journal_user -h localhost -d diploma_database < database/diploma_database_backup.sql

```

> 📌 *Примечание:* Дамп базы данных находится в папке `database/diploma_database_backup.sql`.

### 7. Применение миграций и создание администратора

```bash
# Применение миграций Django
python manage.py migrate

# Создание суперпользователя (если необходимо)
python manage.py createsuperuser

```

### 8. Запуск локального сервера

```bash
python manage.py runserver

```

После запуска проект будет доступен по адресу: [http://127.0.0.1:8000](http://127.0.0.1:8000)

## 🗄️ Структура базы данных

Реляционная модель базы данных включает в себя следующие основные сущности (таблицы):

* **Организационная структура:**
* `Department` — Отделения колледжа
* `Specialty` — Специальности
* `Group` — Учебные группы


* **Пользователи:**
* `Teacher` — Преподаватели
* `Student` — Студенты


* **Учебный процесс:**
* `Discipline` — Учебные дисциплины
* `DisciplinePlan` — Учебные планы дисциплин
* `Classroom` — Кабинеты
* `Schedule` — Расписание занятий
* `Lesson` — Проведенные занятия
* `LessonType` — Типы занятий (лекция, практика, лаб. работа)


* **Контроль знаний и файлы:**
* `Task` — Задания для студентов
* `LessonFile` — Прикрепленные к занятиям файлы
* `Grade` — Оценки студентов
* `Attendance` — Посещаемость
* `AttendanceType` — Статусы посещаемости (присутствовал, Н, Б)


* **Обратная связь:**
* `ContactMessage` — Сообщения обратной связи
* `MessageStatus` — Статусы обработки сообщений

## 🎯 Сценарии использования

### 👨‍🎓 Для студента

| Сценарий | Действие | Ожидаемый результат |
|----------|----------|---------------------|
| **Вход в систему** | Перейти на сайт → нажать "Вход" → ввести логин `student_10_1` и пароль `studentstudent` | Открывается личный кабинет студента |
| **Просмотр оценок** | В ЛК нажать "Мои оценки" | Отображается таблица оценок по всем дисциплинам со средним баллом |
| **Просмотр задолженностей** | В ЛК нажать "Мои задания" → отфильтровать "Несданные" | Список заданий, по которым ещё нет оценки или получена неудовлетворительная оценка |
| **Просмотр расписания** | В главном меню нажать "Расписание" → выбрать группу → нажать "Показать" | Недельное расписание занятий с указанием кабинетов и преподавателей |
| **Просмотр посещаемости** | В ЛК нажать "Моя посещаемость" | Статистика посещаемости и детализация по занятиям (✓ — присутствовал, НП — отсутствовал без причины) |
| **Скачать материалы** | В разделе "Мои задания" → нажать на задание → перейти по ссылке файла | Файл скачивается или открывается в браузере (изображения показываются в предпросмотре) |

### 👨‍🏫 Для преподавателя

| Сценарий | Действие | Ожидаемый результат |
|----------|----------|---------------------|
| **Вход в систему** | Перейти на сайт → нажать "Вход" → ввести логин `teacher_37` и пароль `teacherteacher` | Открывается личный кабинет преподавателя |
| **Просмотр групп и дисциплин** | В ЛК нажать "Мои группы и дисциплины" | Список групп с закреплёнными дисциплинами и количеством проведённых часов |
| **Выставление оценок** | Выбрать группу → нажать "Журнал оценок" → выбрать занятие → нажать на задание → ввести оценки студентам → "Сохранить оценки" | Оценки сохраняются и становятся видны студентам |
| **Отметка посещаемости** | В ЛК выбрать группу → нажать "Журнал оценок" → нажать на дату занятия → сохранить занятие | Для занятия автоматически создаются записи посещаемости со статусом "Присутствовал" |
| **Создание задания** | В журнале оценок → нажать на дату занятия → в разделе "Задания" → "Добавить" → указать название и описание | Задание появляется в журнале и становится доступно студентам |
| **Загрузка материалов** | В карточке занятия → раздел "Файлы" → выбрать файл → "Загрузить" | Файл прикрепляется к занятию, студенты могут его скачать |
| **Экспорт журнала** | В журнале оценок → нажать кнопку "Excel" или "Word" | Скачивается файл с таблицей оценок (формат `.xlsx` или `.docx`) |

| Сценарий | Действие | Ожидаемый результат |

| **Управление дисциплинами** | В панели администратора → "Дисциплины" → "Добавить" → выбрать план, группу и преподавателя | Дисциплина закрепляется за группой и преподавателем |
| **Массовые операции в Django admin** | Перейти на `/admin` → выбрать раздел "Оценки" → отметить записи → выбрать действие "Установить оценку 5" | Выбранным оценкам массово присваивается значение |

### 🗂️ Доступ к стандартной административной панели Django

Помимо кастомной панели администратора, в системе доступна стандартная админ-панель Django:

| Параметр | Значение |
|----------|----------|
| **URL** | `/admin` |
| **Логин** | `admin` |
| **Пароль** | `admin123` |


В стандартной админ-панели доступны:
- Управление пользователями и ролями
- Просмотр и редактирование всех моделей (отделения, группы, студенты, преподаватели и т.д.)
- **Кастомные массовые действия** (выделение группы оценок → установка нужного значения)
- **Кастомные массовые действия** (выделение записей посещаемости → смена статуса)
- **Кастомные массовые действия** (выделение сообщений обратной связи → смена статуса обработки)
- Кнопки "Сохранить и добавить ещё (с заполненными полями)" для быстрого создания связанных записей
